Falando Sobre Formulários

- Introdução

Formulário serve para preenchimento de informações e praticamente trabalha junto com o servidor, para onde é enviado e processado esses dados.
Ele é declarado pela tag <form> , que é composta por atributos para configurações e inputs para compor sua estruta de entrada de informações com os tipos de entradas de dados que cada campo irá aceitar de acordo com o que for configurado.

- Atributos

action

Este atributo é para direcionar para onde as informações do formulário iram, pode ser para o endereço do servidor ou uma página web, URL.

autocomplete

Este atributo possui os valores on e off, onde on é para manter os dados que foram preenchidos no formulário mesmo depois que tenham sido enviados e o off é para inebir esse armazenamento temporário. Por padrão ele já é desabilitado, assim não mantendo as informações que foram digitadas pré salvas

method

Essa atributo possui o método GET e POST:

  • No método GET irar coleta as informações preenchidas no formulário e envia-lás na propria URL do site, isso é perigoso devido ao risco de está enviando dados que são importantes, como senhas, documentos de indentificação e etc.
  • No método POST as informações são capsuladas e enviadas para o link que é atribuido para a tag <action>.

name

É um atributo para nomear a tag e fazer sua indentificação, com isso é possível através do JavaScript fazer uma validação dos dados nos campos que foram inseridos. É necessário para encontrar a instância do formulário.

target

Este atributo está explicado na página sobre links

- Tag Input e Seus Tipos

button

O tipo <button> ele cria um botão na página que podem ser atribuídos eventos e serem programados para controlar funcionalidades customizadas em qualquer lugar de uma página web.
O ideal é que quando tiver um formulário que tenha um botão submit e que a ação dele seja acompanhado de eventos, que esses eventos estejam implementados no cabeçalho da tag.
Os tipos de botões dentro de um formulário atribuido valores para o "type" são:

  • button
    • Apenas configura como um botão clicável, não tem uma ação pré definida. Para executar algum tipo de ação, deve-se configurar os eventos aplicando JavaScript nele.
  • reset
    • Com está configuração ele básicamente limpa os valores dos campos do formulário ao clicar nele, mas apenas do Form que ele estiver dentro da estrutura.
  • submit
    • Está configuração é submitar as informações, finalizar, enviar essas informações.

checkbox

O tipo checkbox é um campo do tipo caixa de seleção, que é permitido selecionar várias opções.
Para garantir que o servidor receba todas as informações selecionadas, é importante que os valores do atributo name seja todos os mesmos nomes e que tenha no final do nome escolhido o símbolo colchetes "[ ]", assim irá garantir que todas as respostas selecionadas sejam enviadas para serem processadas.

color

O tipo color é um campo do tipo cor, ele disponibiliza um botão onde ao clicar irá abrir uma caixa de diálogo* de cor, permitindo selecionar a cor que o usuário desejar.

date

O tipo date é um campo do tipo data que possibilita a digitação no formato de datas e disponibiliza dentro do campo de digitação um botão no canto que ao clicar nele abre uma caixa de diálogo* do tipo calendário com a opção de escolher a data.

email

O tipo email é um campo do tipo e-mail, neste campo os próprios navegadores já possuem um tipo de validação que é pradrão deles, solicitando a necessidade de ter um modelo de endereço correto. Porém neste tipo de campo existe a possibilidade de enviar o formúlario com o campo vazio que não irá gerar nenhum tipo de alerta, mas se o possuir algum texto, ou seja, possível endereço de e-mail que não contenha o "@", irá gerar um alerta solicitando esse complemento.

file

multiple

hidden

name

month

O tipo month é um campo do tipo mês e ano que possibilita a digitação no formato padrão de mês e ano e disponibiliza dentro do campo de digitação um botão no canto que ao clicar nele abre uma caixa de diálogo* do tipo calendário com a opção de fazer a escolha da data que o usuário desejar.

number

O tipo number é um campo do tipo númerico e que aceita tanto números inteiros, decimais e equações literais. Exemplo o "e".
Este tipo de entrada tem a parcularidade de conseguir fixar qual o valor padrão, mínimo, máximo e de quanto em quanto ele acrescenta ou decrescenta o valor, usando respectivamente o atributos value, min, max e o step com valores numéricos como parametros.

password

O tipo password é um campo do tipo senha, que oculta qualquer texto, número ou caracter especial que for digitado nele. Isso é um meio de segurança para prevenir falhas e possibilita a privacidade do usuário.

radio

O tipo radio é um campo do tipo rádio, resumindo melhor, é um campo de seleção de opções. Este tipo de campo permite escolher apenas uma opção dentre as disponíveis.
Para garantir o funcionamento correto, os atributos name de cada input devem estar com os seus valores identicos entre eles, assim irá garantir que apenas um dos botões de opções seja selecionado.

range

O tipo range é um campo do tipo faixa ajustavél, onde possui uma barra que possibilita ajustar para selecionar o valor a desejável.
Assim como no tipo number, ela possui a particularidade de usar atributos para setar parâmetros.

search

submit

selectbox

O tipo selectbox é um campo do tipo caixa de seleção, basicamente é uma lista pré definida dos valores que o usuário pode escolhar, matendo padrões definidos e evitando a necessidade ou possibilidade de digitar algum dado para ser enviado.
Para configurar as opções que o usuário terá disponível para selecionar no selectbox deve-se usar a tag <option> dentro da estrutura de informações da tag <select>. Está tag envia o valor que estiver descrito no atributo value e apresenta para o usuário o valor que está descrito no corpo da estrutura.
Caso seja necessário um tipo de selectbox que o usuário possa selecionar várias opções, existe a tag <multiple> que possibilita essa opção, basta está incluindo ela no cabeçalho da tag <select>.

text

O tipo text é um campo do tipo texto que aceita letras, números e caracteres especiais.

textarea

O tipo textarea é um campo do tipo texto que aceita letras, números e caracteres especiais. A diferença dele para o text é que ele é um campo que não depende da tag <input> para ser estruturado e por padrão ele é maior do que os demais campos, próprio para inserir mais informações e possui opção de redimensionar a área de digitação. Se caso o layout não permite que o usuário fique redimensionando o tamanho da área de digitação, é possível através do CSS travar o tamanho da área.
Algo importante é que o tamanho padrão da área de digitação, pode ser alterado, sendo definindo através dos atributos rows e cols, que é o tamanho da altura e largura, respectivamente, sendo a altura a quantidade de linhas e largura a quantidade colunas.

url

O tipo url é um campo do tipo url que significa Uniform Resource Locator, ou mais conhecido como links, esse campo aceita o preenchimento do endereço do link apenas por completo, caso esteja faltando alguma informação básica no momento em que for salvar o formulário clicando no botão de salvar ou enviar os dados que foram preenchidos ele irá aparecer uma caixa de diálogo* solicitando o prechimento correto.
A abaixo tem as informaçóes e os significados dos campos que são composto por uma url completa, além de informar quais campos são obrigatórios:

  • https://
    • Protocolo**
  • www.
    • Sub domínio**
  • google.com
    • Nome do domínio**
  • /pesquisas
    • Pasta ou página que será acessado

week

O tipo week é um campo do tipo número da semana que possibilita a digitação no formato padrão de número da semana do ano e disponibiliza dentro do campo de digitação um botão no canto que ao clicar nele abre uma caixa de diálogo* do tipo calendário com a opção de fazer a escolha do número da semana da data que o usuário desejar.


*- Cada navegador implementa um tipo de caixa de diálogo diferente.
**- Campos obrigatórios
